# UI 设计描述（推流专用 · 5v5 班赛）

> 适用场景：**仅用于 OBS 推流**，不考虑手机/响应式。所有展示页面均以 **1920×1080** 为基准。
> 设计重点：
> - **BP（英雄选择）全屏**
> - **结算（MVP）全屏**
> - **比赛进行中：保留大面积视频画面，仅叠加「顶栏 + 侧栏」信息**

---

## 1. 总体设计原则

### 1.1 使用原则
- 页面只服务于 OBS 浏览器源，不考虑独立访问体验
- 所有信息围绕「解说讲得清、观众看得懂」
- 游戏画面始终优先级最高

### 1.2 分辨率与比例
- 基准画布：1920×1080（16:9）
- 禁止自适应缩放布局（避免 OBS 中字体异常）
- 所有模块以像素级固定尺寸设计

### 1.3 字体与风格
- 字体：Noto Sans CJK（简体）
- 整体风格：电竞赛事风、深色背景、高对比
- 两队区分：红 / 蓝主色，胜方高亮（金色）

---

## 2. 页面类型与用途划分

| 页面 | 用途 | OBS 场景 | 是否全屏 |
|----|----|----|----|
| BP 页面 | 英雄禁用/选择展示 | BP 场景 | 是 |
| 比赛页面 | 游戏画面 + 信息包装 | 比赛主画面 | 否（叠加） |
| 结算页面 | 本局结果 / MVP | 结算场景 | 是 |

---

## 3. BP 页面（英雄选择）— 全屏

### 3.1 页面定位
- 用于 BP 阶段
- OBS 直接切到该页面，全屏展示

### 3.2 布局结构（1920×1080）

```
┌────────────────────────────────────────────┐
│ 顶栏：比赛名 · BO · 当前局 · 比分            │ 120px
├───────────────┬───────────────┬────────────┤
│ 红队 BP 面板   │   VS / 阶段    │ 蓝队 BP 面板│
│                │   提示         │            │
│                │                │            │
│                │                │            │
│                │                │            │
│                │                │            │
├────────────────────────────────────────────┤
│ 底部提示：已锁定 / 等待发布 / BP 阶段说明   │ 80px
└────────────────────────────────────────────┘
```

### 3.3 红 / 蓝队 BP 面板
- 宽度：各约 720px
- 内容从上到下：
  1. 队名 + 队徽 + 阵营标识
  2. **Ban 区（3ban 或 5ban）**
     - 横排英雄头像
     - 未选用占位灰框
  3. **Pick 区（5v5）**
     - 按位置纵排：上 / 野 / 中 / 射 / 辅
     - 每行：英雄头像（大）+ 英雄名 + 选手名（可选）

### 3.4 状态表现
- Draft（未发布）：
  - 只显示“等待 BP 数据”占位
- Published（已发布）：
  - 正常显示 BP 内容
- Locked（已锁定）：
  - 顶栏右侧显示「已锁定」徽章
  - 页面整体加金色细描边，提示不可更改

## 4. 比赛进行页面（核心：给视频让位）

> 本页面 **不是全屏 UI**，而是 **OBS 叠加层**，用于覆盖在游戏画面上。

### 4.1 核心目标
- **最大化游戏画面区域**
- 只展示必要信息：比分、局数、队名、BP 结果摘要（可选）

### 4.2 安全视频区域（强约束）

- 游戏视频推荐区域：
  - 左侧或中间 **≥ 1400 × 800**（可按投屏方式调整）
- UI 不得侵占该区域

### 4.3 顶栏（必选）

- 尺寸：1920 × 120
- 位置：顶部
- 内容（从左到右）：
  - 红队名 + 队徽
  - 当前大比分（大号）
  - BO 信息 + 当前局号
  - 蓝队名 + 队徽
- 背景：半透明深色（或透明）

### 4.4 侧栏（可选，但推荐）

- 位置：右侧
- 尺寸建议：
  - 宽度：320–360px
  - 高度：视频区域等高

- 内容选项（可配置显示）：
  - 当前局 BP 阵容摘要（仅英雄头像）
  - 关键选手提示（如 MVP 候选）
  - 技术暂停/重开提示

> 侧栏设计原则：**信息密度低、头像大、可快速扫读**

### 4.5 OBS 使用方式
- 游戏画面：窗口捕获 / 采集卡
- UI（顶栏 + 侧栏）：浏览器源叠加
- UI 页面需支持透明背景

## 5. 结算页面（MVP）— 全屏

### 5.1 页面定位
- 单局结束后使用
- OBS 切场景，全屏展示

### 5.2 布局结构（1920×1080）

```
┌────────────────────────────────────────────┐
│ 顶栏：本局结果 · 局号 · 更新后大比分         │ 140px
├────────────────────────────────────────────┤
│                                            │
│           MVP 大卡（核心视觉）              │
│                                            │
├────────────────────────────────────────────┤
│ 对局摘要 / 高光描述 / 下一局提示            │ 160px
└────────────────────────────────────────────┘
```

### 5.3 MVP 大卡设计
- 左侧：英雄头像（200–260px）
- 右侧：
  - MVP 标识 + 选手名（大号）
  - 英雄名
  - K / D / A（加粗）
  - 关键数据标签（2–3 个，如输出占比、参团率）

### 5.4 胜负表现
- 胜方使用主色渐变背景
- MVP 卡描边金色
- 失败方弱化显示，仅在顶栏体现

---

## 6. 动效与切场建议（导播友好）

- BP → 比赛：直接切 OBS 场景（无需动画）
- 比赛 → 结算：
  - MVP 卡 200–300ms 淡入
- 所有动效不得超过 300ms，避免直播卡顿

---

## 7. 可配置 UI 参数（推流相关）

- `banCount`: 3 / 5
- `bestOf`: BO1 / BO3 / BO5
- `showPlayerName`: true / false
- `showSideBar`: true / false
- `transparent`: true / false

---

## 8. OBS 场景对应关系（推荐）

| OBS 场景 | 浏览器源页面 | 说明 |
|----|----|----|
| BP | /view/bp | 全屏 BP |
| 比赛 | /view/overlay | 顶栏 + 侧栏叠加 |
| 结算 | /view/result | 全屏 MVP |
| 暂停 | /view/pause | 技术暂停备用 |

---

## 9. 验收标准（推流视角）
- 游戏画面无遮挡、比例正常
- 顶栏/侧栏在 OBS 中清晰、不抖动
- BP / 结算 全屏信息完整、无裁切
- 浏览器源透明模式正常（无黑底）
